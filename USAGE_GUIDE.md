# 📖 Humble Custom AI Workstation: 상세 사용 가이드 (v1.7.0)

**Humble Custom AI Workstation**은 엔터프라이즈급 자율 에이전트 성능과 강력한 로컬 보안(DLP) 기술을 결합한 통합 AI 명령줄 작업환경입니다. 본 가이드는 시스템의 모든 기능을 코드 레벨에서 상세히 설명합니다.

---

## 🚀 1. 설치 및 시작 (Setup)

### 환경 요구 사항
*   **Python**: 3.10 이상 권장 (macOS/Linux 네이티브 실행 시)
*   **OS**: macOS (sandbox-exec 지원) 또는 Linux (nsjail 권장)
*   **Windows**: **Docker 필수**. 윈도우 네이티브 환경에서는 커널 격리(Sandbox) 기능이 작동하지 않으므로 반드시 도커를 통해 실행하십시오.
*   **.env 설정**: 프로젝트 루트에 `.env` 파일을 생성하고 `GEMINI_API_KEY=your_key`를 입력하십시오.

### 실행 방법
```bash
cd custom-cli
pip install -e .
custom-cli
```

## 🛠️ 2. 핵심 작업 모드 (Working Modes)

`Shift + Tab`을 통해 실시간으로 전환 가능하며, 대시보드 상단에 현재 상태가 표시됩니다.

1.  **Default Mode**: 에이전트와 자유롭게 대화하며 도구(파일 읽기, 셸 실행 등) 사용 시 매번 사용자의 개별 승인을 받습니다.
2.  **Auto-Edit Mode**: 에이전트가 코드를 직접 수정할 수 있는 쓰기 권한이 강화된 모드입니다. 수정 전 시각적 Diff를 제공합니다.
3.  **Plan Mode**: **강력한 읽기 전용 모드**입니다. 파일 쓰기나 시스템 변경 명령 시도 시 코드 레벨에서 물리적으로 차단하여 안전한 코드 분석을 보장합니다.

## ⌨️ 3. 명령어 레퍼런스 (Slash Commands)

### 🤖 모델 및 에이전트 관리
*   **`/mode <agent|chat>`**: 
    *   `agent`: 도구 사용 및 자율 추론이 가능한 백엔드 (Antigravity SDK)
    *   `chat`: 속도가 빠르고 가벼운 대화 전용 백엔드 (GenAI SDK)
*   **`/model <id>`**: 실행 중 모델을 즉시 교체합니다. (예: `gemini-1.5-pro`)
*   **`/agents <persona>`**: `agents.md`에 정의된 페르소나로 전환합니다.
*   **`/group <p1,p2..> <topic>`**: 여러 페르소나가 참여하는 그룹 토의 세션을 시작합니다.

### 🛡️ 보안 및 시스템 제어
*   **`/protect <label> <regex>`**: 새로운 보안 마스킹 패턴을 즉시 추가합니다.
*   **`/sandbox`**: 커널 레벨 샌드박스 격리 기능을 토글합니다.
*   **`/verbose`**: 보안 작동 및 통신 상세 로그를 실시간으로 확인합니다.
*   **`/refresh`**: 설정 파일(`agent_config.yaml`) 및 지침(`GEMINI.md`)을 다시 로드합니다.

### 🧠 지식 및 컨텍스트 관리
*   **`/compress`**: 대화가 길어질 경우 요약을 통해 컨텍스트를 압축하여 토큰을 절약합니다.
*   **`/pin <index>`**: 중요한 메시지를 고정하여 압축/삭제 대상에서 영구 제외합니다.
*   **`/file <path>`**: 특정 파일의 내용을 보안 스캔 후 즉시 지식 베이스에 주입합니다.
*   **`/peek`**: 마지막 셸 명령(`!`)의 전체 출력(STDOUT/ERR)을 다시 확인합니다.

### 💾 세션 및 작업 자동화
*   **`/save <name>` / `/load <name>`**: 대화 히스토리와 보안 매핑 정보를 영구 저장/복구합니다.
*   **`/fork`**: 현재 상태를 복사하여 새로운 독립 세션으로 분기합니다.
*   **`/commit <msg>`**: 현재까지의 코드 변경 사항을 Git에 자동 스테이징 및 커밋합니다.
*   **`/schedule <name> <delay_s> <command>`**: 지정된 시간 후 명령을 자동 실행합니다.

## ⌨️ 4. 생산성 단축키 (Keyboard Shortcuts)

## ⚙️ 5. 고급 설정 가이드

### `agent_config.yaml` 상세 설정
*   **`capabilities`**: 파일 쓰기(`write: false`)를 통해 물리적인 보안 수준을 결정할 수 있습니다.
*   **`policies`**: 특정 위험 명령어(`rm`, `sudo` 등)를 `deny` 처리하여 안전 사고를 예방합니다.
*   **`exclude_paths`**: `.env` 등 에이전트가 절대 접근해서는 안 되는 경로를 정의합니다.

### `masking_config.yaml` 상세 설정
*   정규식을 통해 유출을 방지할 데이터 형식을 정의합니다.
*   치환된 토큰은 `[LABEL_SESSIONID_INDEX]` 포맷을 가지며, 이는 역추적이 불가능하도록 설계되었습니다.

## 💡 6. 실무 활용 팁 (Best Practices)

1.  **점진적 권한 부여**: 처음에는 `Plan Mode`로 코드를 분석하게 하고, 신뢰가 확보되면 `Auto-Edit Mode`로 전환하여 수정을 요청하십시오.
2.  **보안 우선 참조**: 파일 내용이 궁금할 때는 `/file` 명령이나 `@경로` 참조를 사용하십시오. 모든 내용은 로컬 DLP를 거쳐 안전하게 가명화됩니다.
3.  **지능적 요약 활용**: 대화가 20턴 이상 길어지면 `/compress`를 실행하십시오. 핵심 맥락은 유지하면서 응답 속도가 향상됩니다.
